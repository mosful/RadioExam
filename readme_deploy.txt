1.無線電規章與相關法規  229 / 13
2.無線電通訊方法             132 / 13

4.無線電相關安全防護       36 / 1

另外再將題庫中的所有題目，將相似的題型或相同的題目類型分別彙整整理成表格，提升閱讀與記憶效率，這個重點整理請另外做成一個連結放在畫面右上角，


  方法一：Netlify Drop (最簡單，推薦)

  這是最快、最不需要任何技術背景的方法，甚至不用註冊帳號。

   1. 準備資料夾： 確保您的 index.html, style.css, script.js 三個檔案都放在同一個資料夾中。
   2. 開啟網頁： 用您的瀏覽器打開 https://app.netlify.com/drop (https://app.netlify.com/drop)
   3. 拖曳上傳： 直接將您準備好的整個資料夾拖曳到網頁的虛線框中。
   4. 完成！ 網站會立刻發布，並提供給您一個公開的網址 (例如
      https://<隨機名稱>.netlify.app)。您可以將這個網址分享給任何人。

  方法二：GitHub Pages (適合開發者)

  如果您熟悉 GitHub，這是一個非常標準且好用的方法。

   1. 建立倉庫 (Repository)： 在您的 GitHub 上建立一個新的公開倉庫 (例如，命名為 radio-exam)。
   2. 上傳檔案： 將您的三個檔案上傳到這個倉庫中。
   3. 啟用 Pages 功能：
       * 進入該倉庫的 Settings 頁面。
       * 在左側選單找到 Pages 選項。
       * 在 "Branch" 的地方選擇 main (或是您上傳程式碼的分支)，然後點擊 Save。
   4. 取得網址： 等待一兩分鐘後，頁面會顯示您的網站已經發布，並提供一個公開網址 (格式通常是
      https://<您的GitHub帳號>.github.io/<倉庫名稱>/)。

  方法三：Vercel 或 Cloudflare Pages

  這些是更現代化的平台，也非常推薦，它們提供與 GitHub 整合的自動化流程。

   1. 註冊帳號： 到 Vercel (https://vercel.com) 或 Cloudflare Pages (https://pages.cloudflare.com)
      網站，您通常可以直接使用 GitHub 帳號註冊和登入。
   2. 連接 GitHub： 授權平台存取您的 GitHub 帳號。
   3. 選擇專案： 從您的 GitHub 倉庫列表中，選擇您存放考試系統的那個倉庫。
   4. 部署： 平台會自動識別這是一個靜態網站，您通常只需要點擊「Deploy」按鈕即可。
   5. 完成： 部署完成後，平台會提供給您一個公開網址。未來只要您更新 GitHub 倉庫中的程式碼，網站就會自動重新部署。

  總結建議：

   * 如果您想最快最簡單地看到成果並分享，請使用方法一 (Netlify Drop)。
   * 如果您本身有在用 GitHub，並希望未來方便管理和更新，推薦使用方法二 (GitHub Pages) 或方法三
     (Vercel/Cloudflare)。
